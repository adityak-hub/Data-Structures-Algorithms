What is a constructor?
 
A constructor is a special method that runs(call) automatically when an object is created.

•Constructor have the same as class or structure(c++).
•Constructor don't have a return type.(not even void).
•Constructor are only called once, at object creation.
•Memory allocation happens when constructor is called.

why we use constructor how it works ?

You own a car factory.

You have the blueprint:

Now you create a car.

Java creates the object.

But then Java asks:

"What should I put inside this new car?"

Should the color be red?

Should the speed be 120?

Should the engine be started?

Someone has to initialize the object.

That's exactly what a constructor does.

ACCESS MODIFIER

+-----------------+--------------+----------------+-------------------------------+------------------+
| Access Modifier | Within Class | Within Package | Outside Package (Subclass)    | Outside Package  |
+-----------------+--------------+----------------+-------------------------------+------------------+
| Private         | Yes          | No             | No                            | No               |
| Default         | Yes          | Yes            | No                            | No               |
| Protected       | Yes          | Yes            | Yes                           | No               |
| Public          | Yes          | Yes            | Yes                           | Yes              |
+-----------------+--------------+----------------+-------------------------------+------------------+


Easy Notes---
1. Private
   - Accessible only within the same class.

2. Default (No Modifier)
   - Accessible within the same class.
   - Accessible within the same package.
   - Not accessible outside the package.

3. Protected
   - Accessible within the same class.
   - Accessible within the same package.
   - Accessible outside the package only through inheritance (subclass).

4. Public
   - Accessible from anywhere.



Memory Trick
Private   -> Only the same class.

Default   -> Same class + Same package.

Protected -> Same class + Same package + Subclass (outside package).

Public    -> Everywhere.

This--

•this is a reference variable that refers to the current object.
•It is used to access the current object's variables and methods.


•If a method parameter and an instance variable have the same name, use this to refer to the instance variable. If the names are different, this is optional but often used for readability and consistency.

Destructor->
A destructor is a special method that is automatically called when an object is destroyed. It is used to free or clean up resources used by the object.




Does Java have a Destructor?
No. ❌

Java does not have destructors like C++.

Instead, Java uses a Garbage Collector (GC).

When an object is no longer needed, the Garbage Collector automatically frees its memory.

inheritance
Inheritance is when properties and methods of base class(parent class) are passed on to a derived class(child class) it is called inheritance.

types of inheritance--

1. Single Inheritance: One child class inherits from one parent class.
Animal
   │
   ▼
 Fish

2. Multilevel Inheritance:A child class becomes the parent of another class.

Animal
   │
   ▼
 Fish
   │
   ▼
Shark

3. Hierarchical Inheritance: Multiple child classes inherit from the same parent class.
        Animal
      /    |    \
     ▼     ▼     ▼
   Fish  Bird  Dog


4. Hybrid Inheritance:Hybrid inheritance is the combination of two or more types of inheritance in the same program.
✅ A combination of single, multilevel, and hierarchical inheritance is possible in Java.
❌ Hybrid inheritance that requires multiple inheritance of classes is not supported in Java.

Polymorphism in Java
What is Polymorphism?

Definition:Polymorphism is an OOP concept where one method or object can have many forms or behaviors.

Poly = Many
Morphism = Forms

So, Polymorphism = Many Forms.

Simple Example

Think of a person.

At home → Son
At college → Student
At work → Employee

The same person has different roles (forms).

Similarly, in Java, the same method or object can behave differently.

•Types of Polymorphism
There are 2 types of polymorphism in Java.

1. Compile-Time Polymorphism (Static Polymorphism)

It is achieved using Method Overloading.

Definition
When multiple methods have the same name but different parameters(different types or count) and it is not depend on retrun type, it is called method overloading.

The compiler decides which method to call before run the code.


2. Run-Time Polymorphism (Dynamic Polymorphism)

It is achieved using Method Overriding.

Definition
When a Parent and child classes both contain the same function with a different definition(same function hai but kaam alag hai)and it doesn't depend on return type, it is called method overriding.

The method is chosen at runtime.
